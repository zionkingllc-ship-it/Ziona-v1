We need both because they represent different relationships:
- isFollowing: You follow this user (direction: you → them)
- isFollowedBy: This user follows you (direction: them → you)
This determines the button logic:
- isFollowing only → "Unfollow" (you're following them)
- isFollowedBy only (not following) → "Follow back" (they follow you, you don't)
- Both → "Friends" (mutual)
- Neither → "Follow"
The alternative would be to:
1. Fetch the current user's following list and check if each user is in it
2. Make separate API calls to check each relationship
Both approaches are less efficient than having the backend return it directly in the followers/following queries.
